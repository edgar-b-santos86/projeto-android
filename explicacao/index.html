Planejando a Estrutura do Site (Capitulo 17 Aula 3)

Antes de mexer no código é muito importante nós planejarmos como será construído o nosso site.

Para isso precisamos pensar em três itens no nosso planejamento, e são os seguintes:

  1. Layout;
  2. As cores;
  3. As fontes.

Planejando o Layout

O professor Gustavo Guanabara mostrou uma ferramenta muito interessante, que não é 100% gratuita, porém, mesmo assim a parte do serviço que não é paga nos ajudará muito na construção do nosso layout.

Abra o seu navegador e entre no site MockFlow, ao abrir click em singup ou apenas click em login caso já possua uma conta ou entre com sua conta Google.

O MockFlow serve para criar modelos de estruturas de aplicativos, sites, apresentações, documentos, enfim com ele você consegue literalmente desenhar como o seu site vaio ficar, como e onde você vai querer as caixas do e fazendo isso simulando como se estivesse no navegador ou na tela de uma celular por exemplo.

Após termos desenhado como queremos que fique nosso site, partiremos para as cores dele.

Planejando as Cores

Conforme vimos antes o planejamento das cores também é extremamente importante, e para isso temos algumas opções de site para que possamos escolher a melhor paleta de cores para o nosso site.

As seguintes opções de sites abaixo:

  • Adobe Color
  • Paletton
  • Coolors

Nestas três opções conseguimos várias opções para paletas de cores do nosso site.

Planejando as Fontes

Agora que fizemos o layout, e esolhemos as cores, chegou a vez de escolhermos as fontes do site.

Para isso foi indicado que entrássemos no Fonts Google e também o DaFont.

No Fonts Google iremos utilizar a fonte Debas Neue, e no DaFont iremos utilizar a fonte IDroid.

Ao escolhermos a fonte devemos ter cuidado pois algumas não tem todos os glifos (carcteres), pois se escolhermos uma fonte que por exemplo não tenha acentuação ou caracteres especiais.

Fazendo tudo isso estaremos prontos para iniciar os trabalhos em nosso site.

Transformando o Layout em Código (Caítulo 17 Aula 4)

Cabeçalho

Agora o que faremos é transformar o layout em código. Neste caso é interesante deixar o mockflow aberto e dividir a tela, caso não tenha duas telas, e após fazer isso iremos até a pasta do desafio 10 onde baixamos todo material necessário para construção do site, clique com o botão direito e abra com o VSCode.

Vamos começar colocando um <header></header> pois como planejamos em nosso mock essa será nossa primeira caixa.

Cuidado, a caixa para cabeçalho é <header> e não <head>

Como já vimos o HTML e o desenvolvimento de conteúdo semântico dentro de caixas, então a primeira delas dentro da nossa área <body></body> foi o <header></header>.

No <header></header>, nós vamos colocar um título dentro de um <h1></h1>, e logo após colocaremos o que seria, mas não é uma subtitulo, porém, colocaremos isso em um parágrafo, ou, neste caso um <p></p>

Nosso <header></header> ficou assim:

                
                    <header>
                        <h1>Curiosidades de Tecnologia</h1>
                        <p>Tudo aquilo que você sempre quis saber sobre o mundo Tech, em um unico lugar</p>
                    </header>

Área de Navegação

Abaixo do nosso cabeçalho teremos um menu, e este será colocado através do <nav></nav>, que é a tag em caixa para navegação. Dentro dela colocaremos 4 links. Nossa caixa de navegação ficará assim:

                
                    <nav>
                        <a href="#">Home</a>
                        <a href="#">Noticias</a>
                        <a href="#">Curiosidades</a>
                        <a href="#">Fale Conosco</a>
                    </nav>

Colocarmos os links um abaixo do outro, porém, isso foi apenas para organização do código, o posicionamento na pagina ficará um ao lado do outro, o que também pode ser configurado por meio das CSS, e melhorado com JavaScript mas, veremos as CSS ainda neste desafio, já o JavaScript será em um outro curso que eu acredito que seja complementar a esse e para trabalharmos como desenvolvedores WEB.

Ainda dentro do conteúdo article, colocaremos uma tag para conteúdo extra, para isso utilizaremos a tag <aside></aside>

A primeira versão A primeira tentativa de criar um mascote surgiu em 2007 e veio de um desenvolvedor chamado Dan Morrill. Ele conta que abriu o Inkscape (software livre para vetorização de imagens) e criou sua própria versão de robô. O objetivo era apenas personificar o sistema apenas para a a sua equipe, não existia nenhuma solicitação da empresa para a criação de um mascote. Surge um novo mascote A ideia de ter um mascote foi amadurecendo e a missão foi passada para uma profissional da área. A ilustradora Russa Irina Blok, também funcionária do Google, ficou com a missão de representar o pequeno robô de uma maneira mais agradável. [FOTO] A ideia principal da Irina era representar tudo graficamente com poucos traços e de forma mais chapada. O desenho também deveria gerar identificação rápida com quem o olha. Surgiu então o Bugdroid, o novo mascote do Android. [FOTO] A principal inspiração para os traços do novo Bugdroid veio daqueles bonequinhos que ilustram portas de banheiro para indicar o gênero de cada porta. Incluindo o favicon e criando o arquivo CSS

Para finalizar, foi ensinado/revisado para nós na aula 4 do caítulo 17 como incluir o arquivo do icone de favorito e também criamos o arquivo CSS.

Primeiro vamos colocar o icone com o link de favicon, porém, apesar de quando colocarmos o comando ele já mencionar um arquivo de favicon, é necessário informar o caminho onde o arquivo está.

O mesmo acontece com o arquivo CSS, ambos são incluídos com a tag <link>

No caso do site do desafio 10, o código ficou assim:

                
                    <!DOCTYPE html>
                    <html lang="pt-br">
                    <head>
                        <meta charset="UTF-8">
                        <meta name="viewport" content="width=device-width, initial-scale=1.0">
                        <title>Como Surgiu o Android</title>
                    
                        <link rel="shortcut icon" href="imagens/favicon.ico" type="image/x-icon">
                        <link rel="stylesheet" href="estilo/style.css">
                    
                    </head>
